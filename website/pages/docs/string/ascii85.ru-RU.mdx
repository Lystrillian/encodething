## About Ascii85

Ascii85 - это схема кодирования, использующая 7-битные печатные символы ASCII, также известная как Base85.

Ascii85 делит данные на 4 байта каждый и преобразует их в 5 символов ASCII для их представления.

Существует множество вариантов Ascii85, и Encodething поддерживает следующие три типа Ascii85. Оригинальный - btoa, за ним следуют Adobe и Z85.

|  | Резюме |
| --- | --- |
| Z85 | Используется ZeroMQ." Символы \" (обратная косая черта) и "'" (апостроф), требующие экранирования, не используются. |
| Adobe | Используется для кодирования изображений и других данных в файлах Adobe PostScript и PDF (Portable Document Format). Он заключен в символы "\<~" и "\~>". |
| btoa | Форма команды UNIX btoa. В прошлом использовалась для обмена двоичными данными, но больше не встречается. Заключена в строки "xbtoa Begin" и "xbtoa End". |

Символы ASCII, используемые в Ascii85, выглядят следующим образом: рассматривайте 4-байтовое значение как беззнаковое целое число, вычисляйте из него каждый разряд (5 цифр) 85-й десятичной системы, а затем найдите результат преобразования Ascii85 на основе следующих символов ASCII

|  | ASCII символы |
| --- | --- |
| Z85 | 0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ.-:+=^!/\*?&\<>()\[\]{}@%$# |
| Adobe | !"#$%&'()\*+,-./0123456789:;\<=>?@ABCDEFGHIJKLMNOPQRSTUVWXYZ\[\\\]^\_\`abcdefghijklmnopqrstu |
| btoa | !"#$%&'()\*+,-./0123456789:;\<=>?@ABCDEFGHIJKLMNOPQRSTUVWXYZ\[\\\]^\_\`abcdefghijklmnopqrstu  <br />(The original was from " " (space) to "t", but because some mailers excluded trailing spaces, it was later replaced by the "!" to "u" characters without spaces.) |

Например, если мы преобразуем слово "Hello" с помощью Ascii85, оно будет выглядеть следующим образом.

1\. Разделяйте каждые 4 байта; если меньше 4 байт, заканчивайте "00" для подстановки.

```
48656C6C<sub>(16)</sub> 6F000000<sub>(16)</sub>  (Hell o)
```

2\. Каждые 4 байта обрабатываются как беззнаковое целое число с большим порядком следования, и значение преобразуется в каждую цифру 85-й десятичной системы.

```
48656C6C<sub>(16)</sub>
= 1214606444<sub>(10)</sub>
= <b>23</b> * 85<sup>4</sup> + <b>22</b> * 85<sup>3</sup> + <b>66</b> * 85<sup>2</sup> + <b>52</b> * 85 + <b>49</b>
```

```
6F000000<sub>(16)</sub>
= 1862270976<sub>(10)</sub>
= <b>35</b> * 85<sup>4</sup> + <b>57</b> * 85<sup>3</sup> + <b>33</b> * 85<sup>2</sup> + <b>65</b> * 85 + <b>26</b>
```

3\. Преобразуйте каждую цифру десятичной системы 85 в символ ASCII. Если в конце стоит "00", то в случае с Adobe/Z85 эта набивка исключается.

4\. Все символы объединяются в результат преобразования Ascii85. adobe заключен в "\<~" и "\~>", с новой строкой через каждые 80 символов. btoa заключен в "xbtoa Begin" и "xbtoa End" (включая длину данных, контрольную сумму и т. д.), с новой строкой через каждые 78 символов.

|  | Результат конверсии |
| --- | --- |
| Z85 | nm=QNzV |
| Adobe | \<~87cURDZ~> |
| btoa | xbtoa Begin  <br /> 87cURDZBb;  <br />xbtoa End N 5 5 E 42 S 1f9 R a9f |

Было определено несколько других сокращений.

|  | Сокращения |
| --- | --- |
| Z85 | None |
| Adobe | 00000000<sub>(16)</sub> -> z |
| btoa | 00000000<sub>(16)</sub> -> z  <br /> 20202020<sub>(16)</sub> -> y (btoa v4.2 or later)
